# v2.1 → v2.2

## Bottom Line

> TL;DR — One real breaking change: `cancelOrder` now takes an array and returns
> the typed cancel response. Everything else is additive: a new market type,
> richer trading API, USDH on/off-ramp, streams, builder fees at the adapter
> level, more typed WS subscriptions, and zero runtime dependencies after the
> inline `Decimal` replaces `decimal.js`.

- `cancelOrder` signature changed (see [Breaking changes](#breaking-changes))
- New market type: `priceBucket` (recurring multi-bucket price markets)
- New trading methods: `placeOrders`, `modifyOrder`, `scheduleCancel`,
  `splitOutcome`, `mergeOutcome`, `mergeQuestion`, `negateOutcome`
- New WS subscriptions: `subscribeAllMids`, `subscribeActiveAssetCtx`,
  `subscribeSpotAssetCtxs`, `subscribePerpAssetCtx`
- Builder fee + builder address now configurable at `createHIP4Adapter` time
  (still overridable per order)
- USDH on/off-ramp module (Coinbase + Across, mainnet only)
- Streams module: `createPriceFeed`, `createPerpPriceFeed`
- Orderbook utilities: `computeTradeCost`, `computeEstimatedCost`,
  `computePotentialReturn`
- Core ↔ EVM bridging helpers + HYPE spot mark-px helpers
- New events helper: `fetchSettledOutcome(outcomeId)`
- Inline `Decimal` arithmetic — `decimal.js` is no longer a runtime dep

## Breaking changes

### `cancelOrder` array signature

**v2.1**

```typescript
await adapter.trading.cancelOrder({
  marketId: "1758",
  orderId: "12345",
  outcome: "#17580",
});
// returns void
```

**v2.2**

```typescript
const res = await adapter.trading.cancelOrder([
  { marketId: "1758", orderId: "12345", outcome: "#17580" },
]);
// res: HLCancelResponse — inspect res.response?.data?.statuses for per-cancel results
```

The single-param form no longer compiles; wrap each cancel in an array. The
return is now `HLCancelResponse` so callers can read per-cancel statuses
(`{ status: "success" }` or `{ status: { error: "..." } }`).

Empty arrays throw. Auth-not-set still throws (this part is unchanged).

### `cancelAllOrders` removed

The throw-stub was removed entirely. Cancel multiple orders by passing all
their params in a single array to `cancelOrder([...])`.

### FOK is now rejected (was silently mapped to `Ioc`)

```typescript
// v2.1: silently became Ioc (Fill-and-Kill, allows partial fills)
// v2.2: throws "FOK is not supported by Hyperliquid..."
adapter.trading.placeOrder({ ..., timeInForce: "FOK" });
```

HL's `Ioc` is *Fill-and-Kill*, not *Fill-or-Kill*. Mapping FOK to it was
materially wrong — callers asking for all-or-nothing got partial fills they
didn't want. v2.2 rejects FOK explicitly. Use:

- `FAK` for IoC semantics (fill what you can, cancel the rest), or
- `GTC` for resting limits.

If you genuinely need all-or-nothing, place an `FAK` and verify
`result.shares === requestedAmount` in the response.

### `sqrt` and `Decimal.prototype.log` removed from `lib/precision`

The inline `Decimal` class drops two methods that were exported but never
called outside the lib:

- `sqrt()` — gone. No transcendental-precision sqrt is shipped. If you need
  one, depend on a math library externally.
- `.log(base)` — gone. The internal `floor(log10(x))` chains were replaced
  with `Decimal.prototype.floorLog10()` which uses the BigInt digit count
  directly (no transcendentals).

If you imported `sqrt` from `@perps/hip4` or called `.log(...)` on a
`Decimal` instance, you'll see a build error. Both were rare.

## `placeOrders` and `modifyOrder`

```typescript
// Batch placement — single signed action, atomic submission
const result = await adapter.trading.placeOrders([
  { marketId: "1758", outcome: "#17580", side: "buy",  type: "limit", price: "0.55", amount: "20" },
  { marketId: "1758", outcome: "#17581", side: "sell", type: "limit", price: "0.46", amount: "20" },
]);
// result: { success, results: PredictionOrderResult[] }
// per-order failures are surfaced in result.results without affecting the rest.

// Modify a resting order — preserves queue priority on size-only edits
const modified = await adapter.trading.modifyOrder({
  marketId: "1758",
  orderId: "99999",
  outcome: "#17580",
  size: "30",       // size-only edit → keeps queue priority
});
```

`modifyOrder` returns the same shape as `placeOrder` so callers can read the
(possibly new) `orderId`.

## `scheduleCancel` (dead-man's switch)

Registers a future Unix-millisecond timestamp at which Hyperliquid cancels
every open order from the signing agent. Per-agent (a new schedule replaces
the previous), not per-order.

```typescript
// Arm: cancel everything 1 hour from now if I don't reset
await adapter.trading.scheduleCancel(Date.now() + 3_600_000);

// Push the deadline forward (replaces the previous schedule)
await adapter.trading.scheduleCancel(Date.now() + 3_600_000);

// Disarm
await adapter.trading.scheduleCancel(null);
```

Common pattern: arm it just before a HIP-4 market's settlement auction so
resting limits don't get picked off.

## `userOutcome` share conversions

Four L1-signed conversions on the `userOutcome` envelope. They affect spot
balances directly (no order book interaction).

```typescript
// Split X quote tokens into X Yes + X No shares
await adapter.trading.splitOutcome({ outcome: 17580, amount: "100" });

// Merge paired Yes+No back into quote tokens (null = max available)
await adapter.trading.mergeOutcome({ outcome: 17580, amount: null });

// Merge X Yes shares from EVERY outcome of a question into X quote
await adapter.trading.mergeQuestion({ question: 42, amount: "10" });

// Convert X No of one outcome into X Yes of every OTHER outcome under the question
await adapter.trading.negateOutcome({ question: 42, outcome: 17580, amount: "10" });
```

> Wire note: the negate sub-key is `negateOutcome` (matching HL's testnet UI),
> not `negateQuestion` as the docs body's example renders it.

## `priceBucket` market type

Recurring multi-bucket price markets. Parent question carries
`class:priceBucket|underlying:BTC|expiry:20260505-1700|priceThresholds:81015.3,81258.7|period:15m`,
with each child outcome representing one bucket plus a settlement fallback.

```typescript
const buckets = await adapter.events.fetchMarkets({ type: "priceBucket" });

for (const m of buckets) {
  if (m.isFallback) continue;
  console.log(
    `${m.underlying} bucket ${m.bucketIndex}:`,
    `[${m.lowerBound ?? "−∞"}, ${m.upperBound ?? "+∞"})`,
    "expires", m.expiry,
  );
}
```

`getPriceBucketBounds(thresholds, bucketIndex)` is exported as a helper for
deriving bounds when constructing markets manually.

## Builder fee at adapter level

`createHIP4Adapter` now accepts `builderAddress` and `builderFee` (tenths of
a basis point, `0–1000`). When set, every order placed by the adapter carries
the builder field.

```typescript
const adapter = createHIP4Adapter({
  testnet: false,
  builderAddress: "0xYourBuilderAddress",
  builderFee: 100, // 0.1%
});
```

Per-order overrides still work via `placeOrder({ ..., builderAddress, builderFee })`.

> The builder field is attached whenever a `builderAddress` is set, even if
> the fee is `0` — which lets you tag orders with a builder for tracking
> without charging a fee. Omit `builderAddress` entirely to omit the builder
> field.

Builders must be approved separately via `submitBuilderFeeApproval` before
fees can be collected — the helper now ships with the SDK.

## New WebSocket subscriptions

```typescript
adapter.marketData.subscribeAllMids((mids) => { /* every coin's mid */ });

adapter.marketData.subscribeActiveAssetCtx("@1338", (ctx) => { /* spot ctx */ });
adapter.marketData.subscribeSpotAssetCtxs((all) => { /* bulk spot ctxs */ });

adapter.marketData.subscribePerpAssetCtx("BTC", (ctx) => { /* mark, oracle, funding */ });
```

All return an `Unsubscribe` function and share the single WS connection
managed by `HIP4Client`.

## Streams module

Drop-in price feeds backed by `subscribePrice` + a rolling candle history.

```typescript
import { createPriceFeed } from "@perps/hip4";

const unsub = createPriceFeed(
  adapter.marketData,
  "1758",
  (snapshot) => {
    if (!snapshot.ready) return;
    console.log("mid", snapshot.currentMid);
    console.log("last 14 days of 1h candles", snapshot.candles);
  },
  { interval: "1h", lookbackMs: 14 * 24 * 60 * 60 * 1000 },
);

// later
unsub();
```

`createPerpPriceFeed(client, coin, onSnapshot, opts?)` is the perp counterpart
(uses `HIP4Client` directly).

## USDH on/off-ramp

Mainnet-only fiat ↔ USDH ramp via Coinbase + Across.

- **Buy**: fiat → Coinbase onramp → USDC (Arbitrum) → Across counterfactual deposit → USDH (HyperCore)
- **Sell**: USDH (HyperEVM) → Across swap → USDC (Arbitrum) → Coinbase offramp → fiat

See [`examples/usdh-ramp.ts`](../../examples/usdh-ramp.ts) for the full flow.

The ramp adapter throws if instantiated against testnet config.

## Inline Decimal — zero runtime deps

`decimal.js` is no longer a runtime dependency. The replacement is a
self-contained `Decimal` class backed by a BigInt mantissa + integer exponent
(see `src/lib/precision/primitives/_decimal-impl.ts`).

Behavior matches decimal.js for the methods this SDK uses: arithmetic,
comparison, predicates, rounding (HALF_UP / DOWN), `toFixed`, `toString`
(plain + exponential at decimal.js's default `toExpNeg=-7` / `toExpPos=21`
thresholds), `toSignificantDigits`, `toNumber`, `min` / `max` / `sum`.
Construction is exact; operations round HALF_UP at 28 significant figures
just like decimal.js's default config.

Parity is verified by `tests/unit/decimal-parity.test.ts` (54 tests, ~10k
random seeded inputs comparing every used method against decimal.js, which
is kept as a `devDependency` for this purpose).

If you only consumed the SDK's high-level API (`createHIP4Adapter`, market
data, trading), nothing changes for you. If you imported the precision
primitives directly:

- `Decimal` class is identical in surface for everything except `sqrt()` and
  `.log(base)` which were removed (see [Breaking changes](#breaking-changes))
- All functional helpers (`add`, `sub`, `mul`, `div`, `pow`, `abs`, `neg`,
  `compare`, `gt`, `lt`, `clamp`, `roundToStep`, `formatUsd`, etc.) keep
  their existing signatures and behavior

## Other behavioral changes

- **`fetchSettledOutcome(outcomeId)`** added to the events adapter. Returns
  the typed `HLSettledOutcome` for resolved outcomes (or `null` if not yet
  settled).
- **`fetchUserFillsByTime` / `fetchFrontendOpenOrders`** now include
  `dex: "ALL_DEXS"` in their POST bodies (HL added a `dex` filter; this
  preserves the prior "all dexes" behavior).
- **`createHIP4Adapter` return type tightened** to the concrete
  `HyperliquidHip4Adapter` class instead of the `PredictionsAdapter`
  interface, so callers get access to the extended API surface (e.g.
  `trading.scheduleCancel`, `trading.splitOutcome`) without casts. This is
  source-compatible — any code that typed the variable as
  `PredictionsAdapter` keeps working.
- **No barrel wildcards** — `src/index.ts` and `src/adapter/index.ts` now use
  explicit named re-exports. If you imported anything that was never
  intentionally exported, the build will fail; switch to a public name.
